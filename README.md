# QA Automation Journey

> Documenting my journey from Manual QA to QA Automation Engineer.

## About

This repository documents my professional transition from Manual QA to QA Automation Engineer.

The objective is not only to learn automation tools, but also to document the complete learning process, including technical decisions, practical exercises, real projects, mistakes, solutions, and best practices.

This repository will evolve over time, reflecting both my technical growth and the way I organize and build automation solutions.

---

## Philosophy

> "Do not automate for the sake of automation. Automate to improve quality, optimize work, and free up time to analyze, design, and build better testing solutions."
